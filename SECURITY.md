# Política de Seguridad / Security Policy

## Versiones Soportadas / Supported Versions

Solo la última versión publicada recibe parches de seguridad.
Only the latest published version receives security patches.

| Versión / Version | Soportada / Supported |
|---|---|
| 0.2.9 (latest) | ✅ |
| < 0.2.9 | ❌ |

---

## Reportar una Vulnerabilidad / Reporting a Vulnerability

**ES:** Si descubres una vulnerabilidad de seguridad en Baileys, **no** abras un issue público. Envíala por correo electrónico a:

**EN:** If you discover a security vulnerability in Baileys, **do not** open a public issue. Send it via email to:

📧 **isolatedlabs.cn@gmail.com**

### Qué incluir / What to include

- Descripción clara del problema / Clear description of the issue
- Pasos para reproducir / Steps to reproduce
- Versión afectada / Affected version
- Impacto potencial / Potential impact
- (Opcional) Propuesta de solución / (Optional) Proposed fix

---

## Proceso / Process

1. **Reportas** la vulnerabilidad por correo / **You report** the vulnerability via email
2. **Confirmamos** recepción en 48h / **We confirm** receipt within 48h
3. **Investigamos** y trabajamos en un parche / **We investigate** and work on a patch
4. **Publicamos** el parche en una nueva versión / **We publish** the patch in a new release
5. **Divulgamos** el problema después del parche / **We disclose** the issue after the patch

---

## Divulgación Responsable / Responsible Disclosure

Agradecemos a quienes reportan vulnerabilidades de manera responsable. Publicaremos tu nombre (si autorizas) en los agradecimientos de la versión corregida.

We appreciate responsible vulnerability disclosure. Your name will be included in the release acknowledgments (if you authorize).
